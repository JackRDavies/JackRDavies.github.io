Raspberry Pi Projects

During my time at secondary school, I had a desperation to get more into developing software that allowed me to control more than just content on a screen. I wanted to be able to interact with the real world, through hardware and so I did just that. I had entered a competition back in the year of 2013 which was named "The hour of code", which was backed by the Raspberry Pi foundation and Google, to provide schools with 20 Raspberry Pi's.

I was the only student in the entirety of my school to compete and win. All that was needed of me was to take part in an online page that required me to develop some simple programs, which consisted of both simple code and drag/drop code blocks.

The staff at my school were impressed with my achivement and so was I. Out of the 20 units that were assigned to the school, I was allowed to keep 5 of them for myself. Of course, I made good use of them.

You're probably wondering what a Raspbery Pi is, well it's a single board computer that is very energy efficent and can be powered by a mobile phone charger. It features an energy efficient ARM processor, GPIO (General Input and Output) pins, HDMI, audio port, Ethernet and USB ports, making it a complete computer that can be attached to a TV or monitor. It runs Linux, an open-source operating system.

This is what the Raspberry Pi looks like. It's the size of a credit card!

1.) My first robot

With the 5 Pi's I had used them for different projects, one of the first being a robot. I had persued my wish of interacting with hardware and programmed my first robot, which itilised an old toy as the case, an RC car reciever, LED's and a speaker. Here is what the final outcome was:


Sorry about the bad video quality.

This was my very first robotics project and there was a LOT of things for me to learn, such as how to drive motors, figure out what GPIO pin did what and so forth. The even more difficult task was making an easy-to-use user interface that allowed me to interact with the robot more convenientlty. I had initially used a H-Bridge circuit like one of these but later discovered that it was unable to give full power to my motors as it was designed for 12v power, not 5v. As such, I resorted to more "abnormal" methods of driving my motors - that being an RC Car reciever.

I had taken the main board from an RC Car and solderd some IO headers to the top of it, to make it easier to attach and remove wires, making it more modular. After my modifications, I had to attach the motors to the board via the headers I had soldered and run a program named Pi-RC, which allowed me to figure out the modulation that the RC Board responds to, thus allowing me to control the motors via radio waves.

Of course, I wanted to control the robot from practically anything that I have, so I opted into using a webserver, which is where I also gained a stronger interest in website design. This seemed like a daunting task at first as every time I clicked a button on the website, the page would refresh so it wasn't very convenient to use and the robot would just crash into obstacles. I did my research and found an awesome framework named JQuery, which suppored sending HTTP requests without leaving or refreshing the page, which was great!

I also needed a way of connecting my devices to the robot so I purchased an Edimax WiFi adapter from ebay and then configured the system to use it as a hotspot. Of course, this also meant that I had to learn how to configure the Linux networking system to be able to lease IP addresses, allow the devices to interconnect with one another etc. The programs I used to make this all possible were hostapd, dnsmasq and apache2

2.) Pi-Powered server

One of the advantages of a Raspberry Pi is how energy efficent they are. They can operate off of a 5v 1A power supply and use about 2 pense a day, which is fantastic as the Pi can be powered on all the time and not be a concern in terms of how much it costs to run it continuously.

I took advantage of the Pi's size and power consumption and turned it into a personal home file server aswell as a website server and VPN (Virtual Private Network), all of which is configured through the command line interface (CLI). Admittedly the Pi isn't a very powerful device so not a lot should be expected of it, as the one I am using only has a single core ARMv6 700MHz CPU with 512MB of RAM. This just about suits my needs perfectly, never the less.

Want to know how I made it?

Setting up a Raspberry Pi as a server is a fairly straight forward task. For this section, I'll cover the process of installing a basic file and web server. Setting up a VPN is more of a daunting and lengthy task but there is tutorials on the internet that explain it much better than I, such as this.

The webserver

This is really easy to do and takes a matter of minutes if you have a decent internet connection. To begin with, open a terminal on your Raspberry Pi and type the following command:
sudo apt-get update
sudo apt-get install apache2 php5
This may ask to confirm the install operation, if it does just type the letter Y and hit enter. The Pi should download and install the program(s).

The file server

This is a really simple one. The Pi is already hosting a file server but it is part of the SSH service. Simply download a program named Filezilla Client. Once installed onto your personal computer, open Filezilla and enter your Pi's IP address and login credentials, specifying port 22 as the port number, then click "quick connect". Example:
